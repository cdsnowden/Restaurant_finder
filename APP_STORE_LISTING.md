### Notes for Reviewer
This app uses Google Places API to search for restaurants. No sign-in is required. All data is stored locally on the device.

To test the app:
1. Enter any US zip code (example: 10001, 90210, 60601)
2. Optionally select filters (distance, cuisine, price)
3. Tap "Search Restaurants"
4. View results and try "I'm Feeling Lucky" button
5. Tap "Choose This Restaurant" to save a visit
6. Go to "My Visits" tab to see saved restaurants

The app requires an internet connection to search for restaurants but saved visits can be viewed offline.

## When You're On Your Mac - Action Items

### Step 1: Environment Setup
```bash
# Check macOS version
sw_vers

# Install Xcode from Mac App Store (if not installed)
# Then install command line tools:
xcode-select --install

# Verify Flutter
flutter doctor

# Check CocoaPods
pod --version
```

### Step 2: Open Project in Xcode
```bash
cd /path/to/restaurant_finder
open ios/Runner.xcworkspace
```

### Step 3: Configure Signing
In Xcode:
1. Select "Runner" project in left sidebar
2. Select "Runner" target
3. Go to "Signing & Capabilities" tab
4. Enable "Automatically manage signing"
5. Select your Team (Apple Developer account)
6. Verify Bundle Identifier: `com.chris.restaurantfinder`

### Step 4: Build IPA
```bash
# Make sure you're in the project root
cd /path/to/restaurant_finder

# Build for iOS
flutter build ipa

# IPA will be at: build/ios/ipa/restaurant_finder.ipa
```

### Step 5: Upload to App Store Connect
**Option A: Using Xcode**
1. Open Xcode
2. Go to Window > Organizer
3. Select Archives tab
4. Select your archive
5. Click "Distribute App"
6. Select "App Store Connect"
7. Follow prompts

**Option B: Using Transporter App**
1. Download Transporter from Mac App Store
2. Open Transporter
3. Drag and drop your IPA file
4. Click "Deliver"

### Step 6: In App Store Connect
1. Go to https://appstoreconnect.apple.com
2. Select your app
3. Go to version you're preparing
4. Under "Build", click the (+) button
5. Select the uploaded build
6. Complete all metadata fields
7. Click "Submit for Review"

## Common Issues & Solutions

### Issue: "No valid signing identity found"
**Solution**: Ensure you're signed in to Xcode with your Apple Developer account. Go to Xcode > Preferences > Accounts

### Issue: "Provisioning profile doesn't match"
**Solution**: Let Xcode automatically manage signing. Check "Automatically manage signing" in Signing & Capabilities

### Issue: Flutter build fails
**Solution**:
```bash
flutter clean
flutter pub get
flutter build ipa
```

### Issue: "Your account already exists in App Store Connect"
**Solution**: This is normal. Use your existing Apple ID to sign in.
